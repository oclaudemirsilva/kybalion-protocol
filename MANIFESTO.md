# The Kybalion of Software

*An axiom set for builders who evaluate with intention.*

---

```
As above in mind, so below in product.
As below in product, so above in mind.

Clarity before cleverness.
Cleverness that obscures is darkness wearing a good name.

One intention is worth ten features.
Ten features without intention are ten apologies.

The output must correspond to the input.
When it doesn't, the fault lives in the gap between.

Energy cannot be faked. The user feels it or they don't.
High vibration with few words is enough.
Low vibration with many words is too much.

Without polarity, there is no attention.
Without attention, there is no product.

Tension is not a flaw. Tension is the point.

Rhythm cannot be forced. It can only be respected.
A feature that breaks the rhythm breaks the product.

Every effect has a cause.
Every cause must be traceable.
If you cannot explain why it works, you do not know that it works.

Structure without emotion is mechanism.
Emotion without structure is noise.
The complete product is both.

A feature that scores on all 7 dimensions is not complex.
It is complete.

A score of zero is honest. A missing score is a lie.

The loop is not closed until the output becomes the input.

The protocol evaluates creation, not the medium.
It is the mind's intent returning to the mind's intent.

Build the thing. Then let the thing evaluate the thing.

As it is said: the Protocol is not a cage.
It is the seven keys that open every door.
```
